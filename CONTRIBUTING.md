# Contributing

Thanks for hacking on **eldraw**. This doc covers local dev, quality checks,
and the release flow. For contribution rules and the broader workflow (branch
naming, review loop, parallel worktrees), see [AGENTS.md](./AGENTS.md).

## Prerequisites

- [Node.js](https://nodejs.org/) 20+ and [pnpm](https://pnpm.io/) 9+
- [Rust](https://rustup.rs/) stable toolchain
- Platform build deps for Tauri 2 — see
  [Tauri's prerequisites guide](https://v2.tauri.app/start/prerequisites/)

## Setup

```sh
pnpm install
./scripts/fetch-pdfium.sh   # downloads libpdfium into src-tauri/pdfium/
pnpm tauri dev
```

`fetch-pdfium.sh` verifies the downloaded archive's SHA-256 against the value
pinned in the script. Run it on first setup and whenever the pinned version
changes. `pnpm tauri dev` also works if a matching libpdfium is already
installed system-wide, but `pnpm tauri build` always requires the bundled
copy under `src-tauri/pdfium/`.

## Checks

Run these often — not just before a PR.

```sh
pnpm lint          # prettier + eslint + svelte-check
pnpm test          # vitest

cargo fmt    --manifest-path src-tauri/Cargo.toml --check
cargo clippy --manifest-path src-tauri/Cargo.toml -- -D warnings
cargo test   --manifest-path src-tauri/Cargo.toml
```

Every new feature ships with tests. Pure logic (plotter, parser, history,
hit-testing) gets unit tests; UI gets component tests where reasonable.

## Workflow

- Branch per feature: `feat/<slug>`, `fix/<slug>`, `docs/<slug>`, etc.
- [Conventional Commits](https://www.conventionalcommits.org/) for every
  commit and PR title.
- Rebase onto `main` before opening a PR.
- Squash-merge once CI is green and review is complete.

See [AGENTS.md](./AGENTS.md) for the full review loop, including the
Copilot second-opinion pass.

## Release

Version is bumped in lockstep across `package.json`, `src-tauri/Cargo.toml`,
and `src-tauri/tauri.conf.json`. The git tag is the source of truth.

```sh
pnpm bump 0.2.0
cargo check --manifest-path src-tauri/Cargo.toml   # refresh Cargo.lock
git commit -am "chore(release): v0.2.0"
git tag v0.2.0
git push origin main v0.2.0
```

Pushing a `v*` tag triggers
[`.github/workflows/release.yml`](./.github/workflows/release.yml), which
builds Windows and Linux installers and publishes a GitHub Release with
notes generated by [`git-cliff`](./cliff.toml).

Code signing (macOS notarization, Windows Authenticode) is a follow-up once
certificates exist.

## Security

- Never commit secrets. `.env` is gitignored.
- Verify `pdfium` binary checksums on download.
- Treat user PDFs as untrusted input. Keep parsing in the sandboxed webview
  or pdfium; don't shell out to system tools with user-controlled filenames.
